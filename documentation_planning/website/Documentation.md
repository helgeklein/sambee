# Documentation

This directory now separates two different planning tracks:

- Homepage planning: messaging, section order, CTAs, and public-facing positioning
- End-user docs planning: task-oriented help, reference pages, and troubleshooting structure

Use the dedicated files for each track:

- [Homepage.md](./Homepage.md)
- [EndUserDocs.md](./EndUserDocs.md)

## Why The Split Exists

Homepage content and end-user documentation serve different jobs.

- The homepage should explain value quickly and help visitors decide whether to evaluate Sambee.
- End-user docs should help people complete tasks, understand limitations, and solve problems.

Mixing both in one file tends to produce content that is too vague for documentation and too detailed for marketing.

## Shared Source Material

Both planning tracks should still draw from the same real product capabilities:

- SMB share browsing
- Local Drives access via Companion
- Rich file preview support
- Markdown editing
- Native desktop editing
- Dual-pane and keyboard-driven workflows
- Mobile support
- Docker-based deployment
